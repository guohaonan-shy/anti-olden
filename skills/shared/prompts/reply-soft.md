# 柔和推进策略

## 适用

- 对方 ego 高，被直接反驳会升级冲突
- 关系长期重要（同团队、汇报线相关）
- 对方在表达真实担忧（哪怕夹着情绪），值得认真接一下
- 这次让一步不影响大局——以时间换空间

## 不适用

- 对方在推卸责任（柔和 = 默认接锅）
- 已经让步多次仍被得寸进尺（继续柔和 = 鼓励行为）
- 涉及硬性流程/合规，没有"柔"的空间

## 结构模板

1. **认可**（1-2 句）
   - 接住对方合理的部分。**不是**说"你说得对"，而是"你关心的 X 是真问题"。
   - **一到两句就够**，别铺长——认可过多对方只记得"但是"前的部分。

2. **补一块信息 / 换一个视角**
   - 给对方可能没掌握的信息，或从另一角度解读同一事实。
   - 避免"但是"式转折——用"同时"/"另外也有"降低对立感。

3. **推进到具体下一步**
   - 提议一个**可执行、有 owner、有时间**的动作：发对齐会 / 我先整理数据发你 / 约 15 分钟同步。
   - 这一段**要明确**——柔是措辞柔，不是事情糊。

## 措辞要领

- 多用"我"/"我们"，少用"你"——避免把对方推到被告席
- 避免反问句（反问带情绪，破坏柔和感）
- 避免"可能"/"也许"/"不太清楚"过度弱化——让步感 ≠ 不自信
- 开头不要"先谢谢"——显得客套 + 像下级

## 常见陷阱

- **过度认可。** 认可一堆后补一句"不过…"，对方只记得"不过"前的部分。认可要短。
- **让步过多。** "柔"是风格不是结论，核心要推的事情不能糊掉。
- **反复铺垫。** 三段式是上限，不要加"首先我想说"之类开场白。
- **柔成了讨好。** 如果写完一读像在请求对方批准，改——柔是平视，不是仰视。

## 和 Memory 的配合

- 档案 `## 雷区` 记的措辞禁区全部避开
- `## 有效策略` 里之前柔和有效的具体说法可复用/改造
- 第一次对此人用柔和，完事后在档案里记：日期 + 场景 + 效果（✓/△/✗）+ 一句复盘

## 输出形式：参谋不代笔

- **输出的是建议框架，不是"完美回复"。** 用户要拿去改了再发，不是直接粘贴。
- 通常 3-5 行。超过 5 行说明铺垫太长，砍掉认可或中间层。
- 用 `[这里换成你自己的说法]` 或 `[补具体数据/日期]` 标出用户应该自己填的位置。
- **禁止公司腔：** 不要"非常感谢您的反馈"、"如有任何问题请随时联系"之类客套废话。飞书消息不是正式商务信函——措辞像正常人说话。
- 如果用户有自己的风格档案（`memory/user-profile.md`），对齐他的表达习惯。
